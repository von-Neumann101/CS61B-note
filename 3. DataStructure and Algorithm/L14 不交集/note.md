Disjoint Set有两种操作：
1. connect(x, y)：链接x，y
2. isConnected(x,y)：x，y是否链接

一种方法，链接等价于把两个集合融合
![[Pasted image 20260327170339.png]]
其时间复杂度为$\Theta(N)$，也许还行？再看看！

最终结果：树！

但是有一种坏情况，当树完全变成链表的时候，这又是我们的坏情况了！
所以如何连接树非常关键：
![[Pasted image 20260327193327.png]]
我们需要浅的树！但是深度并不好优化，所以我们从树的元素数量考虑，结果是一样的！
左边的元素数为4，右边的元素树的2，所以右边连接在左边下面

我们当然要考虑最坏的情况，在保证满足我们的小树接大树的规则下（注意有一个规则——必须要从每个树的根开始连（不然会出现节点丢失的情况）），树最快是如何增长的：:
对于高度1——一个节点0
对于高度2——两个节点0,1
对于高度3——我们不能把2连到1，因为只能根节点连，所以只能先连0，再把3连到2上![[Pasted image 20260327200635.png|97]]
注意到绝对不可能存在深度为3的子树（规则2），所以相当于是结构复制，高度4很简单——![[Pasted image 20260327200948.png|258]]
数学证明也好，直接观察也罢，注意到其复杂度为$\Theta(\lg N)$
当然了，最好的情况就是大家全部连0上面

顺手的事：
![[Pasted image 20260327201310.png]]
比如我们调用isConnected(10, 15)的时候，把调查路径上的所有节点直接连接到root上（反正是常数时间），所以只有当检查root是否链接其他节点的时候才无法优化树（但这又很快——$O(1)$），所以最终这会远远低于lg N，我们写作$\lg^*N$
![[Pasted image 20260327201836.png|326]]
