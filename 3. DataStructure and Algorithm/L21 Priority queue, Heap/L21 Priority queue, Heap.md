# 优先队列
MinPQ有四个操作
1. add
2. getSmallest
3. removeSmallest
4. size
当然我们也有对应的取最大的操作

背景：我们要运行一天的机器，用于寻找这一天中前M高的高能粒子
使用普通的数组来记录能量大小，这会非常耗内存，有多少个例子，就要耗多少内存
我们想使用固定空间复杂度的数据结构来记录

很简单，我们一旦储存超过M个数据，立刻移除最小的元素
我们学过三种：排序数组，BST，Hash表
前者太慢，显然不行，BST无法处理相同的数据，Hash表没有排序概念，查找最小很慢

我们使用堆
# Heap
## 定义
Binary min-heap是***complete***且 遵循***最小堆性质***的二叉树
最小堆：每个节点的标签都小于等于其子节点标签
complete：每层都是满的，**除了最底层**（最底层要遵循**任何节点的左边不为空**）
