# Dijistra
![[Pasted image 20260409102400.png]]
我们从A开始，我们设置到A最短为0
然后看BC，同时弹出A，由于A到BC是5和1，我们记录**当前**最短路径为B=5，C=1
接下来我们看最近的顶点——C，同时移除C，然后C通向BD，我们发现了C到B只要1,1+1=2<5我们更新B=2
接下来我们到B然后弹出B，到D，发现更短的路径，更新D
![[Pasted image 20260409102855.png]]

![[Pasted image 20260409135447.png]]**Dijistra的核心在于，不会存在“绕远路反而更短”的情况（因为边权非负）**
比如A-C，他是A到C和B之间最小的，如果A-C不是整体最小的话，那么路线一定是A-B-XXX-C，但是由于边权非负，这就要求A-B的边权小于A-C的边权，矛盾！

Dijistra不能用于负边权，这违反了上面的性质

我们每次都要丢节点（也不是每次，就是丢掉当前最小）
我们假设每次都要改distTo（最多E次）


有一个问题，DIjistra算法告诉我去图的所有节点的最短路径，这是没有必要的，因为有时候我们只要到一个地方去，比如我们从北京到合肥，我不需要知道从北京到任何其他地方的最短
同时，这会导致性能降低（时间大量增加）
