 2-3 trees变LLRBs
 ![[Pasted image 20260329114742.png|218]]
 红色线对我们查找没有任何影响，他只影响添加元素

**add**：
我们要做的是：在**不构建2-3树的情况下，利用红色连接模拟其全部结构**
2-3在加入元素时，直接把元素加入叶子节点
![[Pasted image 20260329140948.png|509]]
但是我们之前断言过：所有的红色连接必须朝左，所以我们左转E![[Pasted image 20260329141115.png|89]]
现在考察叶子节点元素超过L的情况：
![[Pasted image 20260329141355.png|553]]
我们暂时允许右倾，现在继续考虑如何实现2-3的中间翻上去
![[Pasted image 20260329141713.png|322]]
规则：
1. add必须用红线
2. 出现单个右倾则把父节点左转
3. 如果有两个连续的左红线，我们右转链接的第一个节点
4. 如果又有左红又有右红则反转颜色
