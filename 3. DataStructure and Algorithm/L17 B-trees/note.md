# BST
BST的最坏情况就是线性时间，新来的节点一直增大（不断往右边叠）变成链表
![[Pasted image 20260328153406.png|565]]
BST的构建和我们add的顺序直接相关（我们最好让BST茂密），一般来说随机添加会较为茂密，高度期望为$4.311\ln N$

注意一下**前驱**指的是所有比x小的节点中，最大的那个，**后继**则是所有比x大的节点中最小的那个
```
	10
   /  \
  5    15
	  /  \
	 12   20
```
对于节点15来说：
1. 前驱=12
2. 后继=20

在删除节点的时候，使用前驱还是后继替换要随机，这才能保证lgN

只有提前拥有所有数据，才能做到随机，但是现实世界中，例如时间戳，就是递增的

L=3的B-trees一般叫2-3-4树
1,2,3,4,5,6,7加入2-3树
![[Pasted image 20260328161332.png]]
**性质**：
1. B-trees的所有叶子节点都和root距离相等，
2. 有k个元素的非叶子节点一定有k+1个子节点

树的深度在$\log_{h+1}N$到$\log_2N$之间
查找的复杂度是$O(\lg N)$
