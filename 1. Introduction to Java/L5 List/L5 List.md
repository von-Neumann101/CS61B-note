## 链表
一个问题，SLList的getLast或者addLast很慢
我们考虑直接维护一个last，指向最后一个节点
![[Pasted image 20260320085152.png]]
但是我们发现由于节点没有储存上一个节点的信息，所以指向他完全找不到上一个节点的任何信息，这导致删除Last很慢（必然的，因为节点的储存不是连续的，他们随机地分布在0/1海中）

所以想要让每个节点都能访问到他的上一个节点，就必须存储一个prev指向上一个节点。此名为——双向链表
![[Pasted image 20260320085658.png]]
DLList -> doubly linked list
SLList -> singly linked list

显然，哨兵节点也需要有两个（还有一个更好的方法，就是只要一个哨兵节点，prev和next都用）

## 数组
内存盒子：申明变量的时候，给我们一个内存盒子用来存储对象
数组是由许多个内存盒子序列组成的特殊对象
数组是固定长度，且每个盒子里必须是相同类型的对象
```java
x = new int[3];
y = new int[]{1, 2, 3};
int[] z = {1, 2, 3}; //这里省略了new关键字，但声明必须在一行
```
`System.arraycopy(b, i, x, j, k)`把b从i开始的k位，复制到x的第j位

**2D数组**：Java实际上没有二维数组，他是保存引用的数组
