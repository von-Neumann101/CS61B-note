# Merge Sort
如果我们已经有两个排序好的数组，只需要Merge Sort就能在线性时间排序
![[Pasted image 20260415085733.png|288]]
每一层要操作N次，一共logN层，所以最终时间复杂度为$\Theta(N\log N)$
每次拆半都要创建新数组，所以空间复杂度为$\Theta(N)$
可以就地排序，但是导致了一个大的系数
# Insertion Sort
把每个元素放到“相对”正确的位置
![[Pasted image 20260415090513.png|405]]
先放32，然后15放到32前，2放15前......
每次插入的时间复杂度为O(N)，这会很耗费时间。同时，空间也会消耗很大

**改进**：
我们进行就地排序
我们把每个元素和左边的元素交换位置，直到其大于等于左边的元素
时间复杂度为：$\Omega(N)$和$O(N^2)$
我们现在有1M个有序元素，然后我们只交换其中两个元素，应该使用哪一种排序方法
——插入排序

我们注意到**每一次swap，都会减少一个逆序数**，由于插入的时间复杂度高的原因主要在于交换。在这里使用插入排序的时间复杂度接近$\Theta(N)$

实际上插入排序的时间复杂度是$\Theta(N+k)$，其中k是逆序数大小
对于小数组(N<15)，插入排序会很快
