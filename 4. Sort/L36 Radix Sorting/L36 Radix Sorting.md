稳定排序，保持相对顺序不变
![[Pasted image 20260417120245.png|550]]
不稳定排序，完全随机
![[Pasted image 20260417120305.png]]
说白了：对于相等的元素，我们不对他的顺序操作

插入排序？
由于我们只对不等的元素进行操作，所以我们没有修改相等的元素的顺序。则这是稳定排序

快速排序？
我们要随机化，这当然没有稳定

在Java的实现中，对于引用类型使用归并排序，而对基本类型使用快速排序（因为引用类型可能有隐藏的值，需要有稳定性）——5和5是一样的，Dog和Dog名字相同，他的其他东西不一定一样

## Counting sort
假设我们需要排序的是无重，从0开始到N的连续自然数
![[Pasted image 20260417144533.png]]
显然5要排在索引为5的地方......

如上是计数排序的一种特殊情况，对于重复元素——计数
![[Pasted image 20260417145209.png]]

推广到抽象情况，我们一共有N个键，词表大小为R（键的可能取值）
时间和空间复杂度都是Theta(N+R)
当R<<N的时候，计数排序不错

比如对城市人口排序，城市可能就30个（N)，但是人数能到百万（R)，不过我们通过四舍五入，如果我的级数单位不是人，而是万人，这就能变成利于计数排序的版本

LSD：我们把计数排序和按位排序结合在一起（按位排序取模所以把R降得很低）
Theta(w*(n+r))
例如w对于整数来说是10（Java int类型上限）
![[Pasted image 20260417151625.png|389]]
我们实际上仍然是按位排序，不过按位排序的具体方法是计数排序

MSD：分桶再分桶，递归下去排序
`["banana", "apple", "bat", "car"]`
我们把abc分桶
```
a: ["apple"]
b: ["banana", "bat"]
c: ["car"]
```
每个桶里继续排（递归或者直接出）
最好情况，Theta(N+R)我们只需要看首位就能排序（R是桶数量）
最坏我们需要一直比较到底，就是w*(n+r)
