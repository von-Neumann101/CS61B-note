方法重载(overloding)：方法名相同参数列表不同
长期来说，我们会出现一些问题：
1. 代码很相似
2. 对于新的参数传入，我们需要添加其他方法
3. 我们发现某个重载的方法出错了，由于代码是相似的，我们必须修改所有的方法

例如我们有一个对于泰迪、哈士奇、土狗......的洗澡方法，我们可以把这些对应的狗向上转型得到狗

在Java中我们同理，使用一个最根基的类型(hypernym)作为我们“重载”的基础：
1. 先创建一个引用类型
2. 再指定其下位为......
